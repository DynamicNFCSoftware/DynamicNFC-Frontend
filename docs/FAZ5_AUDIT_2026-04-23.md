## Section F ‚Äî Risk Flags

- Legacy routes are still actively wired: `App.jsx` still mounts both `/enterprise/crmdemo/dashboard` and `/automotive/dashboard`, so retirement will break direct route entry unless route deprecation happens in lockstep.
- Multiple non-admin pages still link to these legacy dashboards (`Home`, `Developers`, `Enterprise`, `CRMGateway`, several demo portals), so retirement has broad navigation blast radius beyond `App.jsx`.
- Both legacy dashboards rely on localStorage-seeded demo datasets (`dnfc_events`) while `/unified` is tenant/Firestore-driven; behavior parity may drift for demo-only interactions.
- Legacy notification systems use `BroadcastChannel` directly; `/unified` notification behavior is provider/event driven and mode-dependent, so event timing/volume may differ after retirement.
- Legacy dashboards do not import `firestoreTracking` directly, and `/unified` audited files also do not import it directly; if downstream portals still depend on this service, telemetry assumptions should be validated before hard retire.
- Automotive legacy includes fixed ‚ÄúTop Configurations‚Äù and weekly area-trend modules without obvious 1:1 `/unified` UI blocks; these may be perceived as feature loss if expected by stakeholders.
